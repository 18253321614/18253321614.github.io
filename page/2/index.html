linux常用命令

〇、连接远程linux主机

  • 使用ssh命令（linux主机需要开启ssh服务）：ssh <用户名>@<主机IP>，例：ssh root@101.32.155.100
    之后会提示输入用户密码，正确输入密码即可进入。

一、日常使用命令

1、开关机命令

  1. shutdown -h now：现在关机
  2. shutdown -r now：现在重启
  3. reboot：现在重启计算机
  4. su [用户名]：切换用户，不加用户名的话默认切换到root用户
  5. password ：修改密码 # 未成功
  6. logout：用户注销

2、常用快捷键命令

  1. tab：补全
  2. ctrl+l：清屏
  3. ctrl+r：查找历史命令
  4. ctrl+c：终止
  5. ctrl+k：删除此处至末尾的所有内容
  6. ctrl+u：删除此处至开始的所有内容

3、常用工具命令

  1. man

  2. wc（wordcount）：文本统计

    1. 常见参数
      1. -l：只查看行数
      2. -w：只查看单词数
      3. -c：只查看字符数

  3. du：文件大小统计

    1. 常用参数：
      1. -s：只统计该文件目录的大小，不递归
      2. -h：人性化显示单位

  4. find：文件检索命令

  5. grep命令：

    Grep 是 Global Regular Expression Print 的缩写，它搜索指定文件的内容，匹配指定的模式，默认情况下输出匹配内容所在的行。注意，grep 只支持匹配而不能替换匹配到的内容。

  6. I/O重定向：>， 覆盖输出 ；>>，追加输出

  7. 管道：管道是将前一个命令的输出作为后一个命令的输入

  8. xargs：是给其他命令传递参数的一个过滤器，它擅长将标准输入数据转换成命令行参数

二、常用目录或文件操作命令

1、展示目录列表命令ls（list）

  1. ls；ls -a；ls -l；ll；ll -a；ll -h，友好的显示当前目录下的文件信息；pwd

2、切换目录命令cd（change directory）

  1. cd test；cd ..；cd ～；cd -；

3、目录的创建和删除

  1. mkdir test：在当前目录创建test目录
  2. mkdir -p test/a/b：在test目录下的a目录下创建b目录，如果a目录不存在一并创建
  3. rmdir test：删除当前目录下的test目录

4、文件的创建和删除

  1. touch test.txt：创建text.txt文件
  2. rm test.txt：删除test.txt文件，删除前询问用户
  3. rm -f test.txt：直接删除，不询问
  4. rm -r test.txt：递归删除，test目录及其子目录，删除前询问用户
  5. rm -rf test.txt：递归直接删除

5、文件打包或者解压命令

  1. 打包并压缩文件（打包文件.tag，压缩文件.gz），一般情况下，打包和压缩是在一起的，打包压缩后文件后缀为.tag.gz。
    1. 命令：tar -zcvf <指定打包压缩后的文件名> <要打包的文件>
      1. z：压缩
      2. c：打包
      3. v：显示运行过程
      4. f：指定文件名
    2. 注意：指定新文件名要自己加后缀，如tar -zcvf test.txt.tag.gz aa
  2. 解压压缩包
    1. 命令：tar [-xvf] <压缩文件>
      1. x：解压缩
      2. v、f和打包压缩的时候用法一致
    2. 主要：要解压的压缩文件名要写全，如tar -xvf test.txt.tag.gz
  3. zip类型：unzip <文件名>
  4. rar类型

三、linux中的权限问题

文件详情：

因此可以得出，文件的权限问题就是：所有者、同组用户、无关用户对文件的读、写、执行操作的管理。

注意：读权限用数字4表示，写权限用数字2表示，执行权限用数字1表示，同时拥有读写权限则为数字6

  1. 修改文件/或目录的权限：chmod
    1. 例：修改文件test.txt文件的权限为，所有者拥有读写执行权限、同组用户可读可执行，其他用户没有权限，chmod 730 test.txt
  2. 修改文件的所属用户和所属组
    1. chown userName:groupName fileName，修改文件所属用户和所属用户组
    2. chown userName: fileName，修改文件所属用户
    3. chown :groupName fileName，修改文件所属用户组
  3. 用户管理
    1. 查看所有用户：users，或者查看etc/password
    2. 添加用户：useradd <用户名>，useradd -d <指定家目录> <用户名>，useradd -g <组名> <用户名>
    3. 删除用户：userdel <用户名>，userdel -r <用户名>
    4. 查询用户信息：id <用户名>
    5. 修改用户所在组：usermod -g <新组名> <用户名>
  4. 用户组管理
    1. 查看所有：groups，或者查看etc/group
    2. 添加组：groupadd <组名>
    3. 删除组：groupdel <组名>
    4. usermod -aG wheel username. #centos将用户添加到wheel组即可执行sudo

四、文件/文件夹的复制、移动、查看

  1. 复制命令：cp
    1. 语法
      1. cp [-option] source destination
      2. cp [options] source1 source2 .... directory
    2. 选项和参数
      1. -d 若来源档为连结档属性，则复制结档属性而非文件本身
      2. -p 连同文件的属性一起复制过去
      3. -r 递归连续复制，用于目录
      4. -a 相当于-pdr
      5. -f 强制
      6. -i 若目标档已存在，询问用户
      7. -l 复制文件的硬式链接，而不是文件
      8. -s 复制成为符号链接档，快捷方式
      9. -u destination比source旧才升级
  2. 删除命令：rm
    1. 语法
      1. rm [-option] 文件或者目录
    2. 选项与参数
      1. -r -f -i
  3. 移动或改名：mv
    1. 语法
      1. mv [-option] source destination
      2. mv [options] source1 source2 .. directory
    2. 参数
      1. -rfi

五、文件内容查看

  • cat 有第一行开始显示文件
    • -b 显示行号
    • -n 显示行号 包括空行
    • -A = -vET
    • -T tab显示为见 I
    • -v 列出一些看不出来的特殊字符
    • -E 结尾的断行字节$显示出来
  • tac 从最后一行开始显示，cat倒过来写
  • nl 显示行号
    • -b 制定行号显示方式：-b a 忽略空行，-b t 不忽略空行
    • -n 行号的表示方法：-n ln 左边显示，-n rn 右边显示且不加0 ，-n rz 右边显示且加0
    • -w 行号占用的位数
  • more 一页一页的看
    • enter：下一行
    • 空格：下一页
    • b：上一页
    • :f 显示文档名和目前的行数
  • less 一页一页的看，可以翻页
    • B上一页
    • 空格下一页
    • /字串 向上搜索字串
    • ?字串 向下搜索字串
    • n 重复前一个搜索（/ or ?）
    • N反向重复前一个搜索（/ or ?）
  • head 只看头几行
  • tail只看尾几行：tail -n number；-f：表示持续侦测后面所接收的文档名，ctrl+c结束侦测

  1. 编辑sudo命令的配置：visudo

七、磁盘及分区管理

1、Linux磁盘类型

  • IDE类型：
    - 个人电脑常用磁盘类型，缺点是在读写操作时都大量消耗CPU资源 ，优点是性价比高适用面广。
    - 磁盘在系统中的标识：sd，例如第一块IDE类型磁盘的第一个分区为sda1
  • SCSI类型：
    - 服务器常用磁盘类型，缺点是贵，优点是在大量数据读写时，自己的处理器会处理，占用CPU很少的资源；同时扩充性好、稳定、可靠。
    - 磁盘在系统中的标识：hx，例如第一块SCSI类型磁盘的第三个分区为hxa3

2、常用分区类型及特点

  • mbr分区：
    • 最多支持4个主分区
    • 系统只能安装在主分区
    • 扩展分区站一个主分区（扩展分区可有多个逻辑分区）
    • 最大支持2TB，但拥有较好的兼容性
  • gtp分区：
    • 支持无限多个主分区
    • 最大支持18EB的分区1024*1024TB
    • win7 64为以后支持

1、产生原因

跨域问题产生的原因是浏览器的同源策略。浏览器同源策略是浏览器中的一种安全机制，用于防止一个域下的文档或脚本访问另一个域下的文档或脚本。同源指的是两个URL的协议、主机名和端口号都相同，只有这些都相同的情况下，两个文档之间才满足同源条件。

也就是指，当==浏览器地址栏的地址（页面访问地址）==和==接口访问地址==不一样时就会出现跨域问题，浏览器会拦截服务器返回的数据，并报CROS错误。

地质时代简单梳理

发表于 2023-01-30 分类于 求知 本文字数： 475 阅读时长 ≈ 1 分钟

地质时代是指地球上各种地质事件发生的时代。

  1. 从大到小分为宙、代、纪、世、期。比如我们常说的侏罗纪，就是属于显生宙中生代侏罗纪；寒武纪属于显生宙古生代。
  2. 冥古宙、太古宙、元古宙、显生宙，从距今5.41亿年开始就是显生宙了（现在也是），显生宙的标志就是古生代寒武纪生物大爆发。其中前三个时期又称为隐生宙，因为当时生命还很原始。
  3. 将显生宙细看开来，分为古生代-中生代-新生代
    • 古生代：距今2.25亿年-约6亿年前，脊椎海洋生物开始繁盛，藻类、鱼类繁盛，两栖类登录，蕨类植物登录。
      • 寒武纪：5-6亿年前
      • 奥陶纪：4.4-5亿年前
      • 志留纪
      • 泥盆纪
      • 石炭纪
      • 二叠纪
    • 中生代：0.7亿-2.25亿年前，爬行动物的时代，裸子植物
      • 三叠纪
      • 侏罗纪：1.35-1.8亿年前
      • 白垩纪：0.7-1.35亿年前，恐龙灭绝，白垩纪结束，地质进入新生代。
    • 新生代：从7000万年前开始，哺乳动物的时代，被子植物繁盛。
      • 第三纪
      • 第四纪：它从约260万年前开始，一直延续。
        1. 更新世
        2. 全新世：开始于1.2-1万年前，直到现在。
  4. 每个纪的生物都不太一样，而化石是识别地层的头号指标，因此纪是最常用的地址单位。

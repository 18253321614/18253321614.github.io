mysql查询优化

性能下降的原因

  1. 查询语句写的烂
  2. 未使用索引或者索引失效
  3. 关联查询太多
  4. 服务器问题，例如服务器参数设置、线程池、缓冲区等的问题

<–more–>

🏅补充：

MySQL的数据是存储在磁盘文件中的，查询处理数据时，需要先把磁盘中的数据加载到内存中，磁盘IO 操作非常耗时，所以我们优化的重点就是尽量减少磁盘 IO 操作。访问二叉树的每个节点就会发生一次IO，如果想要减少磁盘IO操作，就需要尽量降低树的高度。那如何降低树的高度呢？

假如key为bigint=8字节，每个节点有两个指针，每个指针为4个字节，一个节点占用的空间16个字节（8+42=16）。

因为在MySQL的InnoDB存储引擎一次IO会读取的一页（默认一页16K）的数据量，而二叉树一次IO有效数据量只有16字节，空间利用率极低。为了最大化利用一次IO空间，一个简单的想法是在每个节点存储多个元素，在每个节点尽可能多的存储数据。每个节点可以存储1000个索引（16k/16=1000），这样就将二叉树改造成了多叉树，通过增加树的叉树，将树从高瘦变为矮胖。构建1百万条数据，树的高度只需要2层就可以（10001000=1百万），也就是说只需要2次磁盘IO就可以查询到数据。磁盘IO次数变少了，查询数据的效率也就提高了。

哪些情况需要创建索引:

  1. 主键自动创建唯一索引

  2. 频繁作为查询条件的字段应当建立索引

  3. 查询中与其他表关联的字段，外间关系建立索引

  4. 频繁更新的字段不适合创建索引。

  5. where条件里用不到的字段不创建索引

  6. 单键/组合索引的选择问题（高并发下倾向创建组合索引）

  7. 查询中的排序字段，排序字段若通过索引去访问将大大提高排序速度。

  8. 查询中统计或者分组字段

哪些情况不要创建索引:

  1. 标记路太少
  2. 经常增删改的表
  3. 字段重复率很高的的字段，对字段加索引没什么效果。
  4. 索引选择性：索引列中不同值的数量和总记录数的比值，例如一个表中有2000条数据，索引列有1980个不同的值，这时候索引选择性=0.99，这时候索引的效率就很高。

Select语句的执行过程

基本执行过程

  1. 解析查询：MySQL首先解析SQL查询，检查语法是否正确，确定查询涉及的表和列。
  2. 查询优化：查询优化器会分析查询，选择最有效的执行计划。这可能涉及决定使用哪个索引，以及在多表查询中的表的连接顺序。
  3. 读取数据：根据优化器的决定，MySQL会从磁盘读取必要的数据页到内存中。如果数据页已经在内存中（例如，在缓冲池中），则直接使用。
    • 如果查询使用了索引，MySQL会首先读取索引，然后根据索引找到对应的数据页。
      • 如果没有使用索引，MySQL会进行全表扫描，读取所有的数据页。
  1. 执行查询：在内存中，MySQL执行查询，过滤和处理数据。对于SELECT查询，这可能包括应用WHERE子句的条件，执行连接，分组和排序数据，以及计算聚合函数。
  2. 返回结果：最后，MySQL将结果集返回给客户端。结果集是满足查询条件的行的集合。

示例

以InnoDB为例：

  1. 读取索引表到内存
  2. 索引查找：MySQL首先在索引表中查找满足username='great'的索引项。这个过程通常比全表扫描要快，因为索引结构允许MySQL快速定位到满足条件的索引项（B+树种搜索）。
  3. 行查找（回表）：对于每个满足条件的索引项，MySQL会使用索引项中的行指针（对于非聚簇索引）或主键值（对于聚簇索引）来查找对应的数据行。这个过程可能涉及从磁盘读取数据页到内存。
  4. 行过滤：MySQL在内存中对每个找到的数据行应用age > 18条件进行过滤。只有满足条件的行才会被包含在结果集中。
  5. 结果返回：最后，MySQL将结果集返回给客户端。

请注意，这个过程可能会因为查询的具体内容和数据库的配置而有所不同。例如，如果查询只需要索引中的信息（即，覆盖索引查询），那么MySQL可能不需要查找和过滤数据行。

索引优化分析

mysql服务器中，优化器负责对用户的query请求进行优化。当客户端向MySQL请求一条Query，解析器对请求进行分类，将select请求转发给优化器模块，根据query进行一些分析计算，然后得出最后的执行计划。

优化重点

MySQL常见瓶颈：

  • CPU利用率高，一般发生在数据装入内存或从磁盘上读取数据的时候
  • 一次查询IO次数过多，装入数据远大于内存容量的时候(不停的交换)
  • 服务器性能瓶颈

因此对于查询的优化重点就是降低CPU利用率和I/O次数

Explain执行计划分析

使用explain关键字可以模拟优化器执行sql执行语句，从而知道mysql是如何处理你的mysql语句。进而可以分析sql执行的性能瓶颈。

‼️注意：

一个负责SQL语句通常会经过多次查询，最终将数据组合得到结果。Explain执行计划其实就是将一个SQL语句的多次查询的详情列出来。

使用方式：explain + SQL语句

🏅建议：

  1. 对于单值索引，选择针对当前查询字段过滤性更好的索引
  2. 复合索引将过滤性更好的字段放在前面
  3. 组合索引，尽量包含where子句种更多的字段
  4. 通过分析统计信息和调整query的写法达到选择合适索引的目的

排序优化

避免filesort

order by语句使用索引最左前列活使用where子句与order by子句条件满足最左前列。

例：KEY a_b_c(a,b,c)

‼️‼️‼️‼️‼️注意：

where a=const order by b这种情况，where a=const g > const order by b是不影响复合索引a_b_c生效的。就是说乱入不会影响原有的复合索引。

补充：

  1. filesort的两种排序算法
  • 双路排序：是首先根据相应的条件取出相应的排序字段和可以直接定位行数据的行指针信息，然后在sort buffer 中进行排序。排序后再吧查询字段依照行指针取出，共执行两次磁盘io。
  • 单路算法：是一次性取出满足条件行的所有字段，然后在sort buffer中进行排序。 执行一次磁盘io。（sortbuffer要足够大【通过sort_buffer_size参数设置】，否则还是会变成双路排序）
  1. Mysql如何选择

通过比较系统参数 max_length_for_sort_data的大小和Query 语句所取出的字段类型大小总和来判定需要使用哪一种排序算法。

  • max_length_for_sort_data大于数据总和，使用单路
  • max_length_for_sort_data小，使用多路

关联查询

  • 关联查询，左表会使用全表扫描（所以左表关联字段加索引没有意义），右表的关联字段应该添加索引。
  • 左表作为驱动表，应该使用数据量少的表

注意：

  • 尽量减少join语句中嵌套循环的总次数（join (select * from a join b on~~)），永远用小的结果集驱动大的结果集
  • 优先优化嵌套循环的内部循环
  • 保证被驱动表的join条件字段已被索引
  • 当无法保证被驱动表的join条件字段被索引并且内存充足的条件下，不要吝啬joinbuffer

2、切换目录命令cd（change directory）

  1. cd test；cd ..；cd ～；cd -；

3、目录的创建和删除

  1. mkdir test：在当前目录创建test目录
  2. mkdir -p test/a/b：在test目录下的a目录下创建b目录，如果a目录不存在一并创建
  3. rmdir test：删除当前目录下的test目录

4、文件的创建和删除

  1. touch test.txt：创建text.txt文件
  2. rm test.txt：删除test.txt文件，删除前询问用户
  3. rm -f test.txt：直接删除，不询问
  4. rm -r test.txt：递归删除，test目录及其子目录，删除前询问用户
  5. rm -rf test.txt：递归直接删除

5、文件打包或者解压命令

  1. 打包并压缩文件（打包文件.tag，压缩文件.gz），一般情况下，打包和压缩是在一起的，打包压缩后文件后缀为.tag.gz。
    1. 命令：tar -zcvf <指定打包压缩后的文件名> <要打包的文件>
      1. z：压缩
      2. c：打包
      3. v：显示运行过程
      4. f：指定文件名
    2. 注意：指定新文件名要自己加后缀，如tar -zcvf test.txt.tag.gz aa
  2. 解压压缩包
    1. 命令：tar [-xvf] <压缩文件>
      1. x：解压缩
      2. v、f和打包压缩的时候用法一致
    2. 主要：要解压的压缩文件名要写全，如tar -xvf test.txt.tag.gz
  3. zip类型：unzip <文件名>
  4. rar类型

三、linux中的权限问题

文件详情：

因此可以得出，文件的权限问题就是：所有者、同组用户、无关用户对文件的读、写、执行操作的管理。

注意：读权限用数字4表示，写权限用数字2表示，执行权限用数字1表示，同时拥有读写权限则为数字6

  1. 修改文件/或目录的权限：chmod
    1. 例：修改文件test.txt文件的权限为，所有者拥有读写执行权限、同组用户可读可执行，其他用户没有权限，chmod 730 test.txt
  2. 修改文件的所属用户和所属组
    1. chown userName:groupName fileName，修改文件所属用户和所属用户组
    2. chown userName: fileName，修改文件所属用户
    3. chown :groupName fileName，修改文件所属用户组
  3. 用户管理
    1. 查看所有用户：users，或者查看etc/password
    2. 添加用户：useradd <用户名>，useradd -d <指定家目录> <用户名>，useradd -g <组名> <用户名>
    3. 删除用户：userdel <用户名>，userdel -r <用户名>
    4. 查询用户信息：id <用户名>
    5. 修改用户所在组：usermod -g <新组名> <用户名>
  4. 用户组管理
    1. 查看所有：groups，或者查看etc/group
    2. 添加组：groupadd <组名>
    3. 删除组：groupdel <组名>
    4. usermod -aG wheel username. #centos将用户添加到wheel组即可执行sudo

四、文件/文件夹的复制、移动、查看

  1. 复制命令：cp
    1. 语法
      1. cp [-option] source destination
      2. cp [options] source1 source2 .... directory
    2. 选项和参数
      1. -d 若来源档为连结档属性，则复制结档属性而非文件本身
      2. -p 连同文件的属性一起复制过去
      3. -r 递归连续复制，用于目录
      4. -a 相当于-pdr
      5. -f 强制
      6. -i 若目标档已存在，询问用户
      7. -l 复制文件的硬式链接，而不是文件
      8. -s 复制成为符号链接档，快捷方式
      9. -u destination比source旧才升级
  2. 删除命令：rm
    1. 语法
      1. rm [-option] 文件或者目录
    2. 选项与参数
      1. -r -f -i
  3. 移动或改名：mv
    1. 语法
      1. mv [-option] source destination
      2. mv [options] source1 source2 .. directory
    2. 参数
      1. -rfi

五、文件内容查看

  • cat 有第一行开始显示文件
    • -b 显示行号
    • -n 显示行号 包括空行
    • -A = -vET
    • -T tab显示为见 I
    • -v 列出一些看不出来的特殊字符
    • -E 结尾的断行字节$显示出来
  • tac 从最后一行开始显示，cat倒过来写
  • nl 显示行号
    • -b 制定行号显示方式：-b a 忽略空行，-b t 不忽略空行
    • -n 行号的表示方法：-n ln 左边显示，-n rn 右边显示且不加0 ，-n rz 右边显示且加0
    • -w 行号占用的位数
  • more 一页一页的看
    • enter：下一行
    • 空格：下一页
    • b：上一页
    • :f 显示文档名和目前的行数
  • less 一页一页的看，可以翻页
    • B上一页
    • 空格下一页
    • /字串 向上搜索字串
    • ?字串 向下搜索字串
    • n 重复前一个搜索（/ or ?）
    • N反向重复前一个搜索（/ or ?）
  • head 只看头几行
  • tail只看尾几行：tail -n number；-f：表示持续侦测后面所接收的文档名，ctrl+c结束侦测

七、磁盘及分区管理

1、Linux磁盘类型

  • IDE类型：
    - 个人电脑常用磁盘类型，缺点是在读写操作时都大量消耗CPU资源 ，优点是性价比高适用面广。
    - 磁盘在系统中的标识：sd，例如第一块IDE类型磁盘的第一个分区为sda1
  • SCSI类型：
    - 服务器常用磁盘类型，缺点是贵，优点是在大量数据读写时，自己的处理器会处理，占用CPU很少的资源；同时扩充性好、稳定、可靠。
    - 磁盘在系统中的标识：hx，例如第一块SCSI类型磁盘的第三个分区为hxa3

2、常用分区类型及特点

  • mbr分区：
    • 最多支持4个主分区
    • 系统只能安装在主分区
    • 扩展分区站一个主分区（扩展分区可有多个逻辑分区）
    • 最大支持2TB，但拥有较好的兼容性
  • gtp分区：
    • 支持无限多个主分区
    • 最大支持18EB的分区1024*1024TB
    • win7 64为以后支持

大模型部署

发表于 2026-01-20 分类于 AI 本文字数： 1.8k 阅读时长 ≈ 2 分钟

整体路线

大模型部署首先需要有训练好的模型、推理引擎、服务管理发布平台，常用的推理引擎有vLLM、Hugging Face TGI、Ollama等，其中前两者主要用于企业级模型部署，而Ollama比较适合本地化部署。服务发布管理平台负责管理企业部署的各种模型，例如LLM、Embedding、Rerank等，提供统一的API供其他应用访问，但是Xinference本身也是可以作为推理引擎使用的，并且支持各种模型的推理应用。而Dify在上面的作用是应用开发平台，可以用各种模型的API结合内置的工具快速完成应用构建和开发，和Coze比较相似，只是Coze更适合新手，偏向低代码化。

AutoDL是一个算力市场，可在线租用GPU：https://www.autodl.com/home

主线1

对于企业来说，数据安全至关重要，这也是企业私有化部署大模型服务的主要原因，但是除非企业有自己的机房和私有云系统，最好的方式还是采用云服务厂商的云服务，数据安全方面也有云服务厂商做保障，购买后的与资源也是私有的不用担心泄露。

我们可以使用腾讯云部署Dify的docker容器，首先购买服务器，然后在服务器上安装dokcer，使用腾讯云的镜像加速服务中的镜像源地址，使用docker拉取Dify即可完成部署，默认使用80端口，直接访问即可。

这样Dify服务就快速部署好了。接下来就是在Dify中设置大模型的服务，既可以使用各大厂商提供的API服务，也可以使用在自己服务器上部署的模型服务。

主线2

注意：AutoDL可以开启学术加速，下载国外资源失败时可以

主线2使用一句话概括，就是在AutoDL的服务器上安装Ollama后，使用Ollama命令一键部署大模型服务。

在AutoDL中租赁好服务器后，使用脚本安装Ollama，安装好启动Ollama服务，例如OLLAMA_HOST=27.0.0.1:6006 ollama serve（这里指定了开放端口，AutoDL只开放6006）。接着就可以使用Ollama拉取模型，模型名称可以在Ollama的市场中寻找，模型部署好，我们就完成了大模型服务API的部署，接着在Dify就可以调用AutoDL中部署的Ollama服务了。

主线3

在主线三中Xinfraence作为一个推理框架使用，首先在AutoDL市场中租赁服务器和显卡资源，使用pip命令可以直接安装Xinference后启动服务并暴露端口，然后在Xinference中下载模型即可。最后配置Dify和Xinference连接。

补充

  1. 推理服务引擎（如 vLLM、Ollama）在大模型应用中的作用，可以类比为“餐厅的后厨引擎”或“汽车的发动机”。它的核心使命不是让你“拥有模型”（那是下载器的事），也不是让你“管理模型”（那是管理平台的事），而是如何在有限的硬件资源下，以最快的速度、最低的成本，处理最多的用户请求。

  2. 目前推理引擎用的比较多还有vLLM，如果你正在搭建一个高并发的API服务，追求每秒处理最多的Token，且只关注文本生成，选 vLLM，它是性能的首选，虽然文档未直接列出，但它是构建高性能服务的基石，Xinference和Ollama使用资源较大。

  3. 大语言模型推理（LLM Inference） ：是指利用已经训练好的模型参数，将用户输入的文本（Prompt）转化为模型输出的文本（Response）的计算过程。

  4. Ollama不支持重排序模型

问题

上面集中方式都是直接下载大模型，如果自己使用预训练模型微调了一个模型，该怎么部署

这里以LoRA微调为例，微调完成后会产生一个文件夹，里面包含adapter_config.json 和 adapter_model.bin 等文件，在Ollama中可以直接加载，但是要告诉Ollama使用的基座模型是什么。

如果使用的是全量微调并且对模型架构进行了修改之后在微调得到的模型会复杂一些，但本质都是两个东西，模型架构和权重参数，如果仅仅是修改权重参数，模型架构保持不变，那么各种推理框架都可以快速部署。但是入股修改了模型架构，就需要让推理框架了解你的模型架构。

AI编程

发表于 2026-01-29 分类于 AI 本文字数： 630 阅读时长 ≈ 1 分钟

阶段一：tab coding
辅助编程，代码补全结合智能提示，同时提供一个聊天窗口
代表产品：各种IDE插件，最开始的通义灵码、github Copilot、CodeGeeX

阶段二：Vibe Coding
氛围编程，追求速度和创意
直觉式、对话驱动的编程方式，通常结合cursor等工具。主打跟着感觉走，开发者通过自然语言与AI继续持续对话，快速将模糊的想法转换成可运行的代码。
适合简单的脚本编写、代码补全、快速原型设计。但是一旦项目规模变大，就会遇到代码质量问题，因为AI生成的代码通常不够规范、健壮。
代表产品：cursor、Trae

阶段三：Spec Coding
规范编程，核心是先计划在干活。强调在写代码之前先通过结构化的文档明确需求、设计和技术约束。
先输出产品需求文档，然后进行技术设计，最后列出任务清单（基础构架、数据模型、状态管理、UI组件），这些文档都是spec，
代表产品：Kiro（亚马逊）

在实际开发中，最高效的策略往往是两者的结合，根据项目阶段灵活切换：

  1. 探索阶段 (用 Vibe)：当你不知道需求是否成立，或者想快速验证一个点子时，使用 Vibe Coding 快速“画”出原型14。
  2. 定型阶段 (转 Spec)：当产品方向确定，需要进行工程化落地、团队协作或上线运营时，将核心模块通过 Spec Coding 进行重构和规范化，补齐文档和测试。
  3. 混合模式：即使是使用 Spec Coding 的项目，也可以允许用 Vibe Coding 来探索具体的技术方案或UI细节，验证后再纳入规范体系1。

一、日常使用命令

1、开关机命令

  1. shutdown -h now：现在关机
  2. shutdown -r now：现在重启
  3. reboot：现在重启计算机
  4. su [用户名]：切换用户，不加用户名的话默认切换到root用户
  5. password ：修改密码 # 未成功
  6. logout：用户注销

2、常用快捷键命令

  1. tab：补全
  2. ctrl+l：清屏
  3. ctrl+r：查找历史命令
  4. ctrl+c：终止
  5. ctrl+k：删除此处至末尾的所有内容
  6. ctrl+u：删除此处至开始的所有内容

3、常用工具命令

  1. man

  2. wc（wordcount）：文本统计

    1. 常见参数
      1. -l：只查看行数
      2. -w：只查看单词数
      3. -c：只查看字符数

  3. du：文件大小统计

    1. 常用参数：
      1. -s：只统计该文件目录的大小，不递归
      2. -h：人性化显示单位

  4. find：文件检索命令

  5. grep命令：

    Grep 是 Global Regular Expression Print 的缩写，它搜索指定文件的内容，匹配指定的模式，默认情况下输出匹配内容所在的行。注意，grep 只支持匹配而不能替换匹配到的内容。

  6. I/O重定向：>， 覆盖输出 ；>>，追加输出

  7. 管道：管道是将前一个命令的输出作为后一个命令的输入

  8. xargs：是给其他命令传递参数的一个过滤器，它擅长将标准输入数据转换成命令行参数

二、常用目录或文件操作命令

1、展示目录列表命令ls（list）

  1. ls；ls -a；ls -l；ll；ll -a；ll -h，友好的显示当前目录下的文件信息；pwd

2、切换目录命令cd（change directory）

  1. cd test；cd ..；cd ～；cd -；

3、目录的创建和删除

  1. mkdir test：在当前目录创建test目录
  2. mkdir -p test/a/b：在test目录下的a目录下创建b目录，如果a目录不存在一并创建
  3. rmdir test：删除当前目录下的test目录

4、文件的创建和删除

  1. touch test.txt：创建text.txt文件
  2. rm test.txt：删除test.txt文件，删除前询问用户
  3. rm -f test.txt：直接删除，不询问
  4. rm -r test.txt：递归删除，test目录及其子目录，删除前询问用户
  5. rm -rf test.txt：递归直接删除

5、文件打包或者解压命令

  1. 打包并压缩文件（打包文件.tag，压缩文件.gz），一般情况下，打包和压缩是在一起的，打包压缩后文件后缀为.tag.gz。
    1. 命令：tar -zcvf <指定打包压缩后的文件名> <要打包的文件>
      1. z：压缩
      2. c：打包
      3. v：显示运行过程
      4. f：指定文件名
    2. 注意：指定新文件名要自己加后缀，如tar -zcvf test.txt.tag.gz aa
  2. 解压压缩包
    1. 命令：tar [-xvf] <压缩文件>
      1. x：解压缩
      2. v、f和打包压缩的时候用法一致
    2. 主要：要解压的压缩文件名要写全，如tar -xvf test.txt.tag.gz
  3. zip类型：unzip <文件名>
  4. rar类型

三、linux中的权限问题

文件详情：

因此可以得出，文件的权限问题就是：所有者、同组用户、无关用户对文件的读、写、执行操作的管理。

注意：读权限用数字4表示，写权限用数字2表示，执行权限用数字1表示，同时拥有读写权限则为数字6

  1. 修改文件/或目录的权限：chmod
    1. 例：修改文件test.txt文件的权限为，所有者拥有读写执行权限、同组用户可读可执行，其他用户没有权限，chmod 730 test.txt
  2. 修改文件的所属用户和所属组
    1. chown userName:groupName fileName，修改文件所属用户和所属用户组
    2. chown userName: fileName，修改文件所属用户
    3. chown :groupName fileName，修改文件所属用户组
  3. 用户管理
    1. 查看所有用户：users，或者查看etc/password
    2. 添加用户：useradd <用户名>，useradd -d <指定家目录> <用户名>，useradd -g <组名> <用户名>
    3. 删除用户：userdel <用户名>，userdel -r <用户名>
    4. 查询用户信息：id <用户名>
    5. 修改用户所在组：usermod -g <新组名> <用户名>
  4. 用户组管理
    1. 查看所有：groups，或者查看etc/group
    2. 添加组：groupadd <组名>
    3. 删除组：groupdel <组名>
    4. usermod -aG wheel username. #centos将用户添加到wheel组即可执行sudo

四、文件/文件夹的复制、移动、查看

  1. 复制命令：cp
    1. 语法
      1. cp [-option] source destination
      2. cp [options] source1 source2 .... directory
    2. 选项和参数
      1. -d 若来源档为连结档属性，则复制结档属性而非文件本身
      2. -p 连同文件的属性一起复制过去
      3. -r 递归连续复制，用于目录
      4. -a 相当于-pdr
      5. -f 强制
      6. -i 若目标档已存在，询问用户
      7. -l 复制文件的硬式链接，而不是文件
      8. -s 复制成为符号链接档，快捷方式
      9. -u destination比source旧才升级
  2. 删除命令：rm
    1. 语法
      1. rm [-option] 文件或者目录
    2. 选项与参数
      1. -r -f -i
  3. 移动或改名：mv
    1. 语法
      1. mv [-option] source destination
      2. mv [options] source1 source2 .. directory
    2. 参数
      1. -rfi

五、文件内容查看

  • cat 有第一行开始显示文件
    • -b 显示行号
    • -n 显示行号 包括空行
    • -A = -vET
    • -T tab显示为见 I
    • -v 列出一些看不出来的特殊字符
    • -E 结尾的断行字节$显示出来
  • tac 从最后一行开始显示，cat倒过来写
  • nl 显示行号
    • -b 制定行号显示方式：-b a 忽略空行，-b t 不忽略空行
    • -n 行号的表示方法：-n ln 左边显示，-n rn 右边显示且不加0 ，-n rz 右边显示且加0
    • -w 行号占用的位数
  • more 一页一页的看
    • enter：下一行
    • 空格：下一页
    • b：上一页
    • :f 显示文档名和目前的行数
  • less 一页一页的看，可以翻页
    • B上一页
    • 空格下一页
    • /字串 向上搜索字串
    • ?字串 向下搜索字串
    • n 重复前一个搜索（/ or ?）
    • N反向重复前一个搜索（/ or ?）
  • head 只看头几行
  • tail只看尾几行：tail -n number；-f：表示持续侦测后面所接收的文档名，ctrl+c结束侦测

六、系统常用操作命令

  1. 编辑sudo命令的配置：visudo

    Linux sudo命令以系统管理者的身份执行指令，也就是说，经由 sudo 所执行的指令就好像是 root 亲自执行。

    使用权限：在 /etc/sudoers 中有出现的使用者。

  2. 网络管理：ping、ifconfig

  3. 服务管理：

    1. 服务启动｜关闭｜状态查看｜重启： service service_name start|stop|status|restart
      1. linux自带的所有服务名称在：/etc/init.d/ 下
      2. 如：network（网络）、iptables（防火墙）、mysql（mysql5.5之前叫mysqld）
    2. 设置某服务是否开机自启
      1. 检查：chkconfig iptables --list
      2. 设置：chkconfig iptables on|off

  4. 进程管理：ps

    1. 查看某进程：ps -ef | grep java（如安装了JDK可用jps代替这条命令）
    2. 杀死某个进程：kill -9 pid

  5. 端口管理

    1. netstat：netstat 是网络状态查看命令，既可以查看到本机开启的端口，也可以查看有哪些客户端连接。netstat 命令的基本信息如下。
    2. 参数

      • -a：列出所有网络状态，包括 Socket 程序；

      • -c秒数：指定每隔几秒刷新一次网络状态；

      • -n：使用 IP 地址和端口号显示，不使用域名与服务名；

      • -p：显示 PID 和程序名；

      • -t：显示使用 TCP 协议端口的连接状况；

      • -u：显示使用 UDP 协议端口的连接状况；

      • -I：仅显示监听状态的连接；

      • -r：显示路由表；

    3. 常用：netstat -atunlp

七、磁盘及分区管理

1、Linux磁盘类型

  • IDE类型：
    - 个人电脑常用磁盘类型，缺点是在读写操作时都大量消耗CPU资源 ，优点是性价比高适用面广。
    - 磁盘在系统中的标识：sd，例如第一块IDE类型磁盘的第一个分区为sda1
  • SCSI类型：
    - 服务器常用磁盘类型，缺点是贵，优点是在大量数据读写时，自己的处理器会处理，占用CPU很少的资源；同时扩充性好、稳定、可靠。
    - 磁盘在系统中的标识：hx，例如第一块SCSI类型磁盘的第三个分区为hxa3

2、常用分区类型及特点

  • mbr分区：
    • 最多支持4个主分区
    • 系统只能安装在主分区
    • 扩展分区站一个主分区（扩展分区可有多个逻辑分区）
    • 最大支持2TB，但拥有较好的兼容性
  • gtp分区：
    • 支持无限多个主分区
    • 最大支持18EB的分区1024*1024TB
    • win7 64为以后支持

一、什么是 MCP

MCP（Model Context Protocol）是模型上下文协议，可以理解为：给 AI 助手接“外设”的统一协议。
有了 MCP，Copilot CLI 不只是聊天，还能调用你定义的工具（查天气、查数据库、调内部 API 等）。

一句话：MCP 让模型“会用工具”。

二、MCP 里的三个角色

用“餐厅点餐”类比：

  • mcp-client（服务员）：接收你的请求，把请求转给后厨，再把结果带回来。
    在本文里就是 Copilot CLI。
  • mcp-server（后厨）：真正干活的服务，里面定义了具体工具。
    在本文里就是你写的 server.py。
  • mcp（点餐规则）：服务员和后厨之间的标准沟通协议。
    保证“你点什么、怎么传、怎么回”都一致。

你会看到几种运行/连接方式：

  • [1] Local
    本地托管方式，通常最省事，适合本地开发和个人使用。

  • [2] STDIO
    Copilot 启动一个本地子进程，通过标准输入输出通信。
    这是 Python 脚本型 MCP Server 最常见方式。

  • [3] HTTP
    把 MCP Server 当成 HTTP 服务访问，适合远端部署和网关化接入。

  • [4] SSE
    基于 Server-Sent Events 的流式连接，适合持续推送/流式结果场景。

如何选：

  • 本地脚本开发：优先 STDIO / Local
  • 部署到服务器供多人调用：优先 HTTP / SSE
  • 需要流式推送体验：选 SSE
